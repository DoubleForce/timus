Постановка задачи

Предлагаю слеюующую задачу:
http://acm.timus.ru/problem.aspx?space=1&num=1380

Работа может быть организована следующим образом:

1. Один из участников берёт лидерские обязанности и в коде комментариями указывает, что нужно сделать и как (что как называть).

2. Решение задачи обсуждается и дифференцируется на микрозадачи, которые в свою очередь распределяются между членами группы.
Каждый участник выделяет, что ему необходимо для решения микрозадачи и указывает это в комментариях кода. 
Также указывается краткая характеристика проделанной им работы в виде документации к написанным методам и классам.
Пример:
  //Написан класс Peshka, который включает в себя:
  //Переменные координат объекта Peshka.x и Peshka.y
  //Метод с конструктором peshka(координаты) для создания объектов
  //Метод boolean getAbility(координаты) который возвращает возможность хода по заданным коодинатам.

3. Один пишет, второй тестирует на предмет багов и указывает их в разделе (специальный раздел для багов в github).


P.S. Все примеры условны, варианты работы могут комбинироваться, комментарии могут сокращаться. Вариант 1 не обязывает всех вникать в суть выполнения задачи. Вариант 2 удобен для больших задач. Вариант 3 удобен в случае не трудоёмкого, но сложного 
кода.
